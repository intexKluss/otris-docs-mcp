# V8 JavaScript Engine


## Verwendete Version

Eingebunden wird immer die V8-Version, welche sich in der aktiven
LTS-Version von Node.JS enthalten ist. Aktuell enthalten ist in
DOCUMENTS 6:

- Documents 6.2.*: v12.4
- Documents 6.1.*: v11.3


## Neue Features


### Neue Features in 12.4

- Iterator-Hilfsfunktionen:
const set = new FileResultset("lcmContract"); // from(): Iterator für FileResultset erstellen
const it = Iterator.from(set); // take(): Iteriere nur über die ersten drei Verträge aus dem FileResultset
for (const c of it.take(3)) { // Verträge verarbeiten
} // drop(): Überspringe die ersten drei Verträge und
// iteriere über die restlichen Verträge
for (const c of it.drop(3)) { // Verträge verarbeiten
} // filter(), some(), every(), map(), reduce() analog den
// gleichnnamigen Funktionen für Arrays
it.filter(c => c.lcmStatus === "OPEN"); // Iterator über alle offene Verträge
it.some(c => c.lcmStatus === "OPEN"); // true, wenn min. ein Vertrag offen
it.every(c => c.lcmStatus === "OPEN"); // true, wenn alle Verträge offen sind // find(): 1. offenen Vertrag im FileResultset suchen
const contract = it.find(c => c.lcmStatus === "OPEN"); // forEach(): Für alle Verträge im Iterator eine Funktion ausführen
it.forEach(c => util.out(c.lcmId)); // toArray(): Vertragsmappen des FileResultset als Array
const a = it.toArray();
- Neue Methoden für Set:
const s1 = new Set([1, 2]);
const s2 = new Set([2, 3]); const u = s1.union(s2); // [1, 2, 3]
const i = s1.intersection(s2); // [2]
const d = s1.difference(s2); // [1]
- Gruppierung von Arrays (ab v11.7)
const arr = [1, 2, 3, 4];
// Nach gerade und ungerade Zahlen gruppieren; Ergebnis ist ein Objekt
const grouped = Object.groupBy(arr, x => x % 2 === 0);  // grouped = { true: [2, 4], false: [1, 3] }
// Analog; Ergebnis ist eine Map
const grouped = Map.groupBy(arr, x => x % 2 === 0);
- ArrayBuffer.transfer (ab v11.4)
const buffer1 = new ArrayBuffer(8); util.out(buffer1.detached); // false; // Übertrage Inhalt von buffer1 nach buffer2
// Mit Parameter wird ArrayBuffer vergrößert oder verkleinert
const buffer2 = buffer1.transfer();
// buffer1.byteLength === 0
// buffer2.byteLength === 8
util.out(buffer1.detached); // true; // Ähnlich zu transfer(), Größe von buffer3 kann nicht
// angepasst werden
const buffer3 = buffer.transferToFixedLength();
// buffer3.resizable = false
- JSON.parse mit Source (ab v11.4)
// Zusätzliches context-Objekt bei Reviver-Callback
// mit dem ursprünglichen JSON-String, der den Wert
// repräsentiert (context.source).
const result = JSON.parse("9999999999999991", (key, value, context) => BigInt(context.source));
// result === 9999999999999991n // JSON.rawJSON() und JSON.isRawJSON() für Erzeugen eines
// JSON-Objekt, dessen Inhalt als JSON-String ohne Änderungen
// übernommen wird. Sinnvoll um eigene Serialiser zu schreiben.
const bigIntToRawJSON = (key, val) => typeof val === "bigint" ? JSON.rawJSON(String(val)) : val;
const tooBigForNumber = BigInt(Number.MAX_SAFE_INTEGER) + 2n;
const embedded = JSON.stringify({ tooBigForNumber }, bigIntToRawJSON);
// embedded === '{"tooBigForNumber":"9223372036854775807"}'
- Array.fromAsync analog zu Array.from (ab v12.1)
// Nur in asychronen Funktionen verfügbar oder mit Top-Level await
// in ES6-Modulen. Verhält sich sonst analog zu Array.from(). // Erzeugung von Arrays aus asynchronen Generatorfunktionen.
const asyncIterable = (async function* () { for (let i = 0; i < 5; i++) { yield i; }
})();
const a = await Array.fromAsync(asyncIterable); // Erzeugung von Arrays aus synchronen Maps & Sets.
const a = await Array.fromAsync(new Map([ [1, 2], [3, 4] ])); // Erzeugung von Arrays aus synchronen Iteratoren mit Promises.
const a = await Array.fromAsync( new Set([Promise.resolve(1), Promise.resolve(2), Promise.resolve(3)])
); // Erzeugung aus Array-ähnlichen Objekten
const a await Array.fromAsync({ length: 3, 0: Promise.resolve(1), 1: Promise.resolve(2), 2: Promise.resolve(3)
});
- Promise.withResolvers (ab v11.9)
// Erzeugen eines Promise-Objekts mit resove und reject

// Sinnvoll, um aus Streams ein asynchronen Iterator zu
// machen. In Documents 6 derzeit nicht relevant.
let { promise, resolve, reject } = Promise.withResolvers();
// ...


### Neue Features in 11.3

- Intl.Number API v3 (ab v10.6)
Formatfunktionen für Zahlenbereiche
const nf = new Intl.NumberFormat("de-CH", { style: "currency", currency: "CHF", maximumFractionDigits: 2,
});
const v = nf.formatRange(3, 5));
// v = CHF 3.00–5.00
Neue Rundungsoptionen
let v = new Intl.NumberFormat("de-DE", { style: "currency", currency: "EUR", roundingPriority: "auto", roundingIncrement: 5, trailingZeroDisplay: "auto", stripIfInteger: "auto"
}).format(123.456789));
// v = 124.45 €
Strings als Dezimalzahlen
const nf = new Intl.NumberFormat("en-US", { style: "currency", currency: "EUR",
});
const bi = 1000000000000000110000n;
let v = nf.format(bi + "E-6"));
// €1,000,000,000,000,000.11
Neue Optionen für Vorzeichen
var nf = new Intl.NumberFormat("en", { signDisplay: "negative"
});
nf.format(-1.0);  // -1
nf.format(-0.0);  // 0  (note: "auto" produces "-0" here)
nf.format(0.0);   // 0
nf.format(1.0);   // 1
- findLast and findLastIndex für Arrays und typisierte Arrays (ab v9.9):
util.out([1, 2, 3, 4].findLast((el) => el % 2 === 0));
// → 4 (last even element)
- Intl.Enumeration API (ab v9.9):
util.out(JSON.stringify(Intl.supportedValuesOf('calendar')));

// ['ADP', 'AED', 'AFA', 'AFN', 'ALK', 'ALL', 'AMD', ...] util.out(JSON.stringify(Intl.supportedValuesOf('numberingSystem')));

- Statische Klasseninitialisierung (ab v9.4)
class B { static publicField = "A public value";
}
class C extends B { static #privateField = "A private value"; static { util.out("Will be executed only once, when the class is initialized"); // own var/let/const scope var myVar = "My value"; util.out("Variable value: " + myVar); // this and super refer to the constructor and the super class itself util.out("A inherited value: " + super.publicField); // Access to private static fields and methods is allowed util.out("This is static private: " + this.#privateField); } static { // More than one block is allowed util.out("Will be executed only once, after the first block"); }
}
