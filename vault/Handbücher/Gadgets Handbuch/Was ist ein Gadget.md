# Was ist ein Gadget?

Ein Gadget erweitert individuell und mandantenabhängig die Funktionalität von DOCUMENTS. Ein Gadget kann an verschiedenen Stellen in DOCUMENTS integriert werden. Ein Gadget kann sowohl mit anderen Gadgets als auch über eine clientseitige API mit DOCUMENTS interagieren.

Da Gadgets in Portal Skripten implementiert werden, steht bei der Verwendung von Gadgets die Portalscript-API in vollem Umfang zur Verfügung. Außerdem bleiben die Gadgets bei einem Versionsupdate kompatibel.


## Was ist ein Gadget nicht?

Ein Gadget ist kein Ersatz für bereits vorhandene Programmfunktionen von DOCUMENTS. Ein Gadget kann nicht verwendet werden um Berechtigungen in DOCUMENTS zu umgehen oder zu verändern.


## Wo können Gadgets verwendet werden?

Gadgets können an folgenden Stellen in DOCUMENTS verwendet werden:

- Als Schaltflächen im Hauptmenü
- Bei Klick auf auf die Übersicht
- Bei Klick auf die Schaltfläche "Neue Mappe"
- Als Kachel in einem Dashboard (siehe Dashboard-Dokumentation)
- Nach dem Hochladen von Dokumenten über die QuickDropZone
- Als benutzerdefinierte Aktion (an Mappen, Registern und Ordnern)
- An öffentlichen Ordnern
- Als Eigenschaft am Mappentyp
Am Mappendeckel
In der Seitenleiste von Mappen
Im Monitor
- Am Register einer Mappe
- Als Mappen-Feld (Typ: Gadget)
- Als User-Exit an einem Mappen-Feld
- Als Email-Exit für einen Mappentyp

Die verschiedenen Möglichkeiten der Integration sind im Kapitel [Integration von Gadgets](../integration/integration-overview.html) beschrieben.
