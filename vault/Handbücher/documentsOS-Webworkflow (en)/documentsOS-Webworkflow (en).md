# Introduction

With the **documentsOS web workflow**, authorized users can create new workflows or edit existing workflows directly in the web client. Apart from the configuration settings shown under [Deployment](install.html), no further local installations are required at the respective workstation.

The web workflow replaces the previous component for workflow processing (Microsoft Visio with workflow plug-in). During the development of the web workflow, care was taken to ensure that the functions of the previous tool were mapped as congruently as possible, but there are changes and extensions - the following documentation describes all the most important work areas and elements of the web workflow in addition to general information on workflow creation.

Important information on converting existing workflows that were created with Microsoft Visio and the workflow plugin can be found in the chapter [Migration from Visio](migration.html).
