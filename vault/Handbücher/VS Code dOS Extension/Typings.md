# Typings

VS Code provides IntelliSense for third party modules and libraries via tpye declaration (typings) files. So, we can use these files to provide IntelliSense for most of the documentsOS APIs. Read here how you can get the typings for documentsOS and how you can improve your IntelliSense.


## Table of contents

- Available Typings
- Configure Typings
- Using JSDoc
- TS-Check


## Available Typings

- portalScripting.d.ts
The common PortalScripting API (available for any documentsOS version).
- fileTypes.d.ts
The file types on the connected server. Can only be generated by the command Install Typings. If file types on the server changed, call it again.
- scriptExtensions.d.ts
The script lists.
- gadgetApi.d.ts
- documentsClientSdk.d.ts


## Configure Typings

The command **Install Typings** can be used to get suitable typings and configuration files for the connected server (in launch.json).

Or you can enable it on your own:

1. Choose suitable type definition files from here (But you will not get the type declaration files for the file types on your server here! These files can only be generated by the command Install Typings)
2. Paste it to a folder called typings in your Visual Studio Code project folder.
3. If your project root folder does not contain the file jsconfig.json, create one like this:


### jsconfig.json


```json
{
    "compilerOptions": {
        "lib": ["es2016", "scripthost"],
        "target": "es2016"
    },

    "include": [
        "src/**/*",
        "typings/**/*"
    ]
}
```

### Hints for jsconfig.json

- The compilerOptions as shown above are required to avoid conflicts with the default JavaScript Typings. (See Compiler Options or jsconfig Options for more details about compilerOptions. This issue contains information about the conflicts).
- If you use a different folder structure than the one shown here you need to change the include paths accordingly.
- A jsconfig.json in a folder indicates that the folder is the root of a JavaScript project. As a consequence you have to be careful with variable names. Meaning, if there are two variables with same name (e.g. myDocFile) then the IntelliSense for this variables will not work correctly.
- See jsconfig.json reference


## Using JSDoc


### context.createFile

If you created the `fileTypes.d.ts` by executing **Install Typings**, you can get IntelliSense for the file types just by using the `context.createFile` function.


```javascript
var docFile = context.createFile('FileType1');
```

var docFile = context.createFile('FileType1');After using this function `docFile` will be of type `FileType1` and you will get the valid field names in the suggestions.

When inserting the parameter for `createFile` you can simply type `Ctrl + Space` to get the suggestions of valid file type names.

![Ctrl + Space](../assets/contextCreateFile.png)


### context.file

Another way to get IntelliSense for a `DocFile` object is adding JSDoc comments to your DocFile objects. You will need this, if you get the file type object by using `context.file`.


```javascript
/**
 * @type {FileType1}
 */
var docFile = context.file;
```

/**
 * @type {FileType1}
 */
var docFile = context.file;
### Function Parameters

If you write a function with parameter from type `DocFile`, you should also add a JSDoc comment to the function. Then you will get IntelliSense for the file type inside the function.


```javascript
/**
 * @param {Filetype1} docFile
 */
function myFunction(docFile);
```

/**
 * @param {Filetype1} docFile
 */
function myFunction(docFile);
## TS-Check

Add `// @ts-check` at the top of your JavaScript file to get optional JavaScript checking and `// @ts-ignore` to let single lines be ignored by the check.
