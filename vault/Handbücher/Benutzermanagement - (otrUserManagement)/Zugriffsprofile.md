# Zugriffsprofile

Die Anzeige und Bearbeitung von Zugriffsprofilen erfolgt über den Konfigurationsordner **Zugriffsprofile**.
In der Liste stehen die notwendigen Aktionen zum *Erstellen* und *Löschen* von Profilen bereit.
Über die Suchfunktion können einzelne Einträge gezielt gesucht werden.


![profile-1](assets/profile-1.png)

Abb. 14 - Liste Zugriffsprofile

**Hinweis:** Die Liste kann per *Callback* angepasst werden, siehe dazu die [Callback-Dokumentation](callbacks.html).

Per Klick auf einen Listeneintrag können die Details zu einem Profil angezeigt werden.
Werden Einstellungen im Formular geändert, sind diese Änderungen per *Speichern* bzw. *Zurücksetzen* zu bestätigen, diese Aktionen werden bei Änderungen automatisch aktiviert.
In der Detailansicht werden die wichtigen Einstellungen fachlich gruppiert auf verschiedenen Registern dargestellt:


![profile-2](assets/profile-2.png)

Abb. 15 - Zugriffsprofile - Allgemeine Einstellungen


![profile-3](assets/profile-3.png)

Abb. 16 - Zugriffsprofile - Mitglieder


![profile-4](assets/profile-4.png)

Abb. 17 - Zugriffsprofile - Experteneinstellungen

**Hinweis:** Für die wichtigsten Einstellungen stehen im Formular Tooltips als Entscheidungshilfe bereit.

**Hinweis:** Die Detailansicht kann per *Callback* angepasst werden, siehe dazu die [Callback-Dokumentation](callbacks.html).

Mit der Standard-Aktion **Zugriffsprofil umbenennen** kann der *Profilname* umbenannt werden, standardmäßig ist der Profilname bei vorhandenen Zugriffsprofilen im Formular schreibgeschützt. Im Dialog muss der neue Profilname angegeben werden:


![profile-5](assets/profile-5.png)

Abb. 18 - Zugriffsprofil umbenennen

Mit der Standard-Aktion **Zugriffsprofil duplizieren** kann ein vorhandenes Zugriffsprofil mit allen Einstellungen dupliziert werden. Im Dialog muss der neue Profilname angegeben werden:


![profile-6](assets/profile-6.png)

Abb. 19 - Zugriffsprofil duplizieren
