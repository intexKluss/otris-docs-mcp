# Einleitung

Framework für die Entwicklung von Tests in DOCUMENTS:

- Schreiben von Unit-Tests
- Scripting-Funktionen zum Prüfen erwartetem Verhalten (Werte, Exceptions usw.)
- Scripting-Funktionen zum Bereitstellen von Testdaten (Mappentypen, Vorgänge usw.)
- Scripting-Funktionen zum Aufräumen der Testdaten nach Testausführung
- Formate zur Darstellung und Auswertung der Testergebnisse (Text, JUnit)
- CLI für die Ausführung der Tests
- Automatische Ermittlung der Pfadabdeckung (Bericht)


## Verwendung


```sh
$> documents test --help
```
